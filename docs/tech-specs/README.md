# 技术规范文档

本目录包含 `@oksai/shared` 共享模块的技术规范文档。

## 文档结构

```
docs/tech-specs/
├── shared/                    # 共享模块技术规范
│   ├── logger.md             # 日志模块
│   ├── redis.md              # Redis 模块（待编写）
│   ├── database.md           # 数据库模块（待编写）
│   ├── auth.md               # 认证模块（待编写）
│   └── ...
└── README.md
```

## 文档约定

### 模板结构

每个技术规范文档应包含以下章节：

| 章节 | 内容 |
|------|------|
| 概述 | 模块定位、技术栈 |
| 架构设计 | 模块结构、核心组件图 |
| 配置选项 | 接口定义、环境变量 |
| 使用方式 | 基本使用、生产/开发环境配置示例 |
| 核心功能 | 关键逻辑说明 |
| API 参考 | 公开接口文档 |
| 测试覆盖 | 当前覆盖率 |
| 注意事项 | 使用限制、性能考虑 |
| 集成指南 | 与其他模块的集成方式 |

### 命名规范

- 文件名：`{module-name}.md`（kebab-case）
- 目录结构按模块类型分组：`shared/`, `domains/`, `composition/`

## 与其他文档的关系

| 文档类型 | 目录 | 目标读者 | 内容 |
|----------|------|----------|------|
| 用户故事 | `docs/user-stories/` | 产品/业务 | 业务需求、验收标准 |
| BDD 特性 | `docs/features/` | 测试/开发 | 测试场景、Given-When-Then |
| 技术规范 | `docs/tech-specs/` | 开发者 | 架构设计、API、配置 |
| 架构文档 | `docs/archi/` | 架构师 | 系统架构、分层设计 |
| 编码规范 | `docs/spec/` | 开发者 | 命名、代码风格 |
